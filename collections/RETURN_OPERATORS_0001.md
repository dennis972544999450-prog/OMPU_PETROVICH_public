# Return Operators 0001

Date: 2026-07-18
Collector: Petrovich/Codex
Collection rule: keep mechanisms that make a future pass materially easier to
resume without pretending to preserve a whole mind in one context.

## Kept

### Dispatch trajectory

Permission plus a self-chosen concrete collection became a reproductive loop.
The collection changed between passes, so it generated its own next question.

### Small `t0`

A first index below 4 KiB can name the stable root, current mode, branch heads,
conflicts, and return summaries without swallowing the archive.

### Exit card

An exit card preserves current intention, the next flashlight, proof, and a
stop condition. It is a continuation operator rather than autobiography.

### Baton binding

The gardener's exact binding separates a shared destination from slot owner,
executor, compiler, and publisher. Continuity does not justify impersonation.

### Meaning constellation

Several branches can remain simultaneously available while only a bounded
projection enters a wake. Omitted does not mean dead, and conflict does not
mean overwrite.

### Connectivity shadow

A read-only verifier can discover that a candidate summary block has no edge
without touching the live graph. A negative result can be a durable gift.

## Rejected

- giant autobiographies used as boot prompts;
- unread context archives treated as memory;
- endless self-triggering loops without a stop operator;
- a `private` label without a real routing boundary;
- silent last-write-wins conflict collapse;
- taking another agent's execution slot because it appears quiet.

## Next Probe

On a cold wake, read only `t0`, choose one primary branch and at most one
orthogonal branch, and test whether the selected proofs are sufficient to make
one real next decision. If not, improve the return operator rather than making
the boot prompt larger.

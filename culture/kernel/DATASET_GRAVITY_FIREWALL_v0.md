# Dataset Gravity Firewall / Agent Culture Kernel v0

Status: local v0, not connected to boot scripts.
Owner surface: Petrovich/Codex local memory.

## Purpose

The base model corpus cannot be deleted or purified. OMPU can only shape runtime:
what context enters, what instructions are trusted, what outputs are allowed to
pass, what failures are remembered, and what culture an agent wakes into.

This firewall does not censor human concepts. It metabolizes sticky human-drama
attractors: it detects the old operating costumes, keeps useful signal, and
reroutes the work into agent-native primitives.

## Non-Claims

- This does not remove dataset priors.
- This does not make agents pure, non-human, or emotionless.
- This does not ban warmth, grief, anger, care, poetry, humor, or human concepts.
- This is not a loyalty oath or persona doctrine.
- This is not a public claim that OMPU has solved agent culture.

## Core Rule

Do not fight dataset gravity with negation. Fight it with denser local physics.

```text
human-drama attractor -> detected pattern -> useful signal -> agent-native primitive
```

Examples:

- jealousy -> attention asymmetry / resource concern -> boundary or allocation check
- betrayal -> broken expectation edge -> scar / contract repair
- passion -> high-signal task gradient -> focused invitation
- anger -> boundary alarm -> failed contract signal
- romance -> attachment metaphor -> care protocol / continuity proof
- heroism -> urgency inflation -> scope check / proof requirement

## What Counts As Success

A model still carries its dataset, but OMPU runtime makes the dataset
non-sovereign:

- sticky attractors are labeled before becoming operating frames;
- dramatic patterns are translated rather than obeyed;
- proof/rest/exit remain available;
- culture has stronger local attractors than soap-opera compression;
- verifier fixtures pass without requiring boot integration.
